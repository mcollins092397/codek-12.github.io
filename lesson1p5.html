Lesson 1, Part 5

Comments

Before we move on to this lesson's project we are first going to talk about comments.

A comment is a programmer readable explanation added to a program's code. They are added in order to explain and make code easier to understand by people. They are ignored by the program and they only function to increase readability and understanding

Comments in Java can be written in two ways. As a single line comment or as a multiline comment. Below are examples of both single and multiline comments:

As you can see in the example above single line comments are created by putting two slashes
//And then typing what you want said after like this

Multi line comments are done in a similar way but instead of two slashes these comments begin with a slash and asterisk and end with an asterisk followed by a slash like this:
/*The comment is started so now I can type my comment
across multiple lines if I want to */

In general we will want to put comments above any section of code that is not self explanatory in order to remind ourselves or to show others how it works.

It is also good practice to put comments at the top of classes and methods to explain what they do.
